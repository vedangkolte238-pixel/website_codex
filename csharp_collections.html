C# Collections

Understanding Collections in C#

What are Collections?

Collections in C# are classes that store and manage groups of objects. They provide more flexibility than arrays, including dynamic sizing and built-in methods.

Types of Collections

1. List

A List is a dynamic array that can grow or shrink in size. It is part of System.Collections.Generic.

using System; using System.Collections.Generic; class Program { static void Main() { List fruits = new List(); fruits.Add("Apple"); fruits.Add("Banana"); fruits.Add("Orange"); foreach (var fruit in fruits) { Console.WriteLine(fruit); } } }

2. Dictionary

A Dictionary stores key-value pairs. It allows fast lookup by key.

using System; using System.Collections.Generic; class Program { static void Main() { Dictionary students = new Dictionary(); students.Add(1, "John"); students.Add(2, "Alice"); Console.WriteLine(students[1]); // Output: John } }

3. Queue

A Queue represents a first-in, first-out (FIFO) collection.

using System; using System.Collections.Generic; class Program { static void Main() { Queue queue = new Queue(); queue.Enqueue("First"); queue.Enqueue("Second"); Console.WriteLine(queue.Dequeue()); // Output: First } }

4. Stack

A Stack represents a last-in, first-out (LIFO) collection.

using System; using System.Collections.Generic; class Program { static void Main() { Stack stack = new Stack(); stack.Push("One"); stack.Push("Two"); Console.WriteLine(stack.Pop()); // Output: Two } }

5. HashSet

A HashSet stores unique elements and does not allow duplicates.

using System; using System.Collections.Generic; class Program { static void Main() { HashSet numbers = new HashSet(); numbers.Add(1); numbers.Add(2); numbers.Add(2); // Duplicate ignored foreach (var num in numbers) { Console.WriteLine(num); } } }

Advantages of Collections

  • Dynamic sizing
  • Built-in methods for adding, removing, searching, and sorting
  • Better performance for complex data operations
  • Type safety with generic collections

Conclusion

Collections are essential in C# for managing groups of objects efficiently. They are more flexible than arrays and are widely used in real-world applications.
