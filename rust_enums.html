Rust Enums

Defining a Type with Multiple Variants

What is an Enum?

An enum in Rust is a type that can have one of several possible variants. Enums are useful for modeling data that can take on a limited set of values.

Basic Enum Example

enum Direction { Up, Down, Left, Right, } fn main() { let move1 = Direction::Up; match move1 { Direction::Up => println!("Moving up"), Direction::Down => println!("Moving down"), Direction::Left => println!("Moving left"), Direction::Right => println!("Moving right"), } }

Enums with Data

Variants can also hold associated data:

enum Message { Quit, Move { x: i32, y: i32 }, Write(String), ChangeColor(i32, i32, i32), } fn main() { let msg = Message::Move { x: 10, y: 20 }; match msg { Message::Quit => println!("Quit"), Message::Move { x, y } => println!("Move to ({}, {})", x, y), Message::Write(text) => println!("Text: {}", text), Message::ChangeColor(r, g, b) => println!("Color: ({}, {}, {})", r, g, b), } }

Applications

  • Representing a fixed set of choices (like states, directions, commands)
  • Safe pattern matching on different data types
  • Modeling complex data structures like AST nodes or messages
  • Combining with structs for more structured data

Conclusion

Enums in Rust provide a powerful way to define types with multiple variants, enabling clear, safe, and expressive code. When combined with pattern matching, they are a cornerstone of idiomatic Rust programming.
