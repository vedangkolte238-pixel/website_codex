Kotlin Null Safety

Preventing Null Pointer Exceptions in Kotlin

What is Null Safety?

Null safety is a Kotlin feature that helps prevent NullPointerException (NPE) errors. Kotlin handles null values at compile time instead of runtime.

Nullable and Non-Nullable Types

Non-Nullable Type

var name: String = "Kotlin" // name = null ❌ Not allowed

Nullable Type

var name: String? = "Kotlin" name = null // Allowed

Safe Call Operator (?.)

The safe call operator executes code only if the value is not null.

val length = name?.length println(length)

Elvis Operator (?:)

Provides a default value when the variable is null.

val len = name?.length ?: 0 println(len)

Not-Null Assertion (!!)

Forces Kotlin to treat a value as non-null. Use carefully, as it can throw an exception.

val size = name!!.length

Safe Cast (as?)

Safely casts a value and returns null if casting fails.

val value: Any = "Hello" val str: String? = value as? String

Let Function

Executes code only when the value is not null.

name?.let { println("Length is ${it.length}") }

Why Kotlin Null Safety is Important

  • Reduces runtime crashes
  • Makes code safer and cleaner
  • Detects errors at compile time
  • Improves app stability

Conclusion

Kotlin null safety is one of its strongest features. By using nullable types and safety operators, you can write robust and crash-free applications.
