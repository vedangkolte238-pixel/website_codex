SQL Keys

Primary, Foreign, and Unique Keys in Databases

What are SQL Keys?

Keys are constraints that uniquely identify records and establish relationships between tables. They help maintain data integrity.

1. Primary Key

Uniquely identifies each record in a table. Cannot be NULL and must be unique.

CREATE TABLE employees ( emp_id INT PRIMARY KEY, emp_name VARCHAR(50) );

2. Foreign Key

Creates a link between two tables. Ensures that the value in one table exists in another.

CREATE TABLE orders ( order_id INT PRIMARY KEY, customer_id INT, FOREIGN KEY (customer_id) REFERENCES customers(customer_id) );

3. Unique Key

Ensures that all values in a column are unique. Can have multiple unique keys in a table.

CREATE TABLE users ( user_id INT, email VARCHAR(100) UNIQUE );

4. Composite Key

A primary or unique key made up of multiple columns.

CREATE TABLE student_courses ( student_id INT, course_id INT, PRIMARY KEY (student_id, course_id) );

5. Candidate Key

Columns that can qualify as a primary key. A table can have multiple candidate keys.

6. Alternate Key

A candidate key that is not chosen as the primary key.

Conclusion

SQL keys are vital for data integrity, avoiding duplicates, and establishing relationships. Understanding keys is essential for designing relational databases.
