SQL Indexes

Improve database query performance

What is an Index?

An index is a database object that improves the speed of data retrieval. It works like an index in a book, allowing quick access to rows in a table.

1. Creating an Index

Use the CREATE INDEX statement to create an index on a column.

CREATE INDEX idx_student_name ON students(name);

2. Unique Index

Ensures that all values in the indexed column are unique.

CREATE UNIQUE INDEX idx_unique_email ON users(email);

3. Composite Index

An index on multiple columns to optimize queries using multiple columns.

CREATE INDEX idx_name_age ON students(name, age);

4. Dropping an Index

Use the DROP INDEX statement to remove an index.

DROP INDEX idx_student_name;

Benefits of Indexes

  • Speeds up SELECT queries
  • Helps with sorting using ORDER BY
  • Enforces uniqueness when using UNIQUE indexes
  • Reduces full table scans for large tables

Conclusion

SQL indexes are essential for improving database performance. However, too many indexes can slow down INSERT, UPDATE, and DELETE operations. Use them wisely.
