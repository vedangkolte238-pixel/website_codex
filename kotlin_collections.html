Kotlin Collections

Working with Data Groups in Kotlin

What are Collections?

Collections are used to store, manage, and manipulate groups of related data. Kotlin provides powerful and easy-to-use collection APIs.

Types of Collections in Kotlin

  • List
  • Set
  • Map

1. List

A List stores elements in order and allows duplicate values.

val fruits = listOf("Apple", "Banana", "Orange") println(fruits[0])

Mutable List

val numbers = mutableListOf(1, 2, 3) numbers.add(4) println(numbers)

2. Set

A Set stores unique elements and does not allow duplicates.

val cities = setOf("Delhi", "Mumbai", "Delhi") println(cities)

Mutable Set

val ids = mutableSetOf(101, 102) ids.add(103)

3. Map

A Map stores data in key-value pairs.

val student = mapOf(1 to "Rahul", 2 to "Amit") println(student[1])

Mutable Map

val scores = mutableMapOf("Math" to 90) scores["Science"] = 95

Common Collection Functions

  • filter()
  • map()
  • forEach()
  • size
  • contains()
val nums = listOf(1, 2, 3, 4, 5) val even = nums.filter { it % 2 == 0 } println(even)

Conclusion

Kotlin collections make data handling simple, safe, and expressive. They are heavily used in real-world Kotlin applications.
