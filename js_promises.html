JavaScript Promises

Handle Asynchronous Operations

What is a Promise?

A Promise is an object that represents the result of an asynchronous operation. It can be completed in the future or fail.

Promise States

  • Pending – operation is not completed yet
  • Fulfilled – operation completed successfully
  • Rejected – operation failed

Creating a Promise

let myPromise = new Promise(function(resolve, reject) { let success = true; if (success) { resolve("Task completed"); } else { reject("Task failed"); } });

Using then()

myPromise.then(function(result) { console.log(result); });

Using catch()

myPromise.catch(function(error) { console.log(error); });

Using then() and catch() Together

myPromise .then(function(result) { console.log(result); }) .catch(function(error) { console.log(error); });

Promise with setTimeout

let delayPromise = new Promise(function(resolve) { setTimeout(function() { resolve("Data loaded after 2 seconds"); }, 2000); }); delayPromise.then(function(message) { console.log(message); });

Promise Chaining

let numberPromise = new Promise(function(resolve) { resolve(10); }); numberPromise .then(function(value) { return value * 2; }) .then(function(value) { return value + 5; }) .then(function(result) { console.log(result); });

Why Use Promises?

  • Handle asynchronous code easily
  • Avoid callback hell
  • Improve code readability
  • Used in API calls and fetch

Conclusion

Promises make asynchronous programming simpler and cleaner. They are widely used in modern JavaScript applications.
