SQL Transactions

Ensuring reliable database operations

What is a Transaction?

A transaction is a sequence of SQL operations performed as a single unit of work. Either all operations succeed, or none are applied, ensuring data integrity.

ACID Properties

  • Atomicity: All operations succeed or all fail.
  • Consistency: The database remains in a valid state before and after the transaction.
  • Isolation: Transactions do not interfere with each other.
  • Durability: Once committed, changes are permanent.

1. Starting a Transaction

Use BEGIN TRANSACTION to start a transaction.

BEGIN TRANSACTION; UPDATE accounts SET balance = balance - 500 WHERE account_id = 1; UPDATE accounts SET balance = balance + 500 WHERE account_id = 2; COMMIT;

2. Commit

Saves all changes made by the transaction permanently.

COMMIT;

3. Rollback

Undoes all changes made in the transaction if an error occurs.

ROLLBACK;

4. Savepoints

Allows partial rollback within a transaction.

SAVEPOINT sp1; UPDATE accounts SET balance = balance - 100 WHERE account_id = 1; ROLLBACK TO sp1;

Conclusion

SQL transactions ensure data integrity and consistency. They are essential when performing multiple related operations in a database.
