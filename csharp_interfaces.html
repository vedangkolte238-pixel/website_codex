C# Interfaces

Understanding Interfaces in C#

What is an Interface?

An interface in C# is a contract that defines a set of methods, properties, or events that a class must implement. It does not contain any implementation itself.

Why Use Interfaces?

  • Define a contract for classes to follow
  • Achieve abstraction in your code
  • Support multiple inheritance of behavior
  • Promote loose coupling and flexibility

Defining an Interface

Syntax:

interface IShape { void Draw(); double CalculateArea(); }

Implementing an Interface

Example of a class implementing the interface:

using System; interface IShape { void Draw(); double CalculateArea(); } class Circle : IShape { public double Radius { get; set; } public void Draw() { Console.WriteLine("Drawing Circle"); } public double CalculateArea() { return 3.14 * Radius * Radius; } } class Program { static void Main() { Circle circle = new Circle(); circle.Radius = 5; circle.Draw(); Console.WriteLine("Area: " + circle.CalculateArea()); } }

Key Points

  • Interfaces cannot contain fields or constructors.
  • All methods in an interface are implicitly public and abstract.
  • A class can implement multiple interfaces.

Advantages of Interfaces

  • Facilitates code reusability and flexibility.
  • Supports multiple inheritance in C# without ambiguity.
  • Helps to design loosely coupled systems.

Conclusion

Interfaces are essential in C# for designing clean and maintainable code. They allow you to define contracts and ensure consistent behavior across different classes.
