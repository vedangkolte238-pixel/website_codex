Ruby Modules

Reusable Code Containers in Ruby

What is a Module?

A module in Ruby is a collection of methods, constants, and class variables that can be mixed into classes. Modules provide namespaces and support mixins, allowing code reuse without using inheritance.

Defining a Module

module Greetings def say_hello(name) puts "Hello, #{name}!" end def say_goodbye(name) puts "Goodbye, #{name}!" end end

Including a Module in a Class

class Person include Greetings end p = Person.new p.say_hello("Alice") p.say_goodbye("Alice")

Module as a Namespace

Modules can also be used to group related classes and constants to avoid name conflicts.

module MathTools PI = 3.14159 class Circle def self.area(radius) PI * radius * radius end end end puts MathTools::Circle.area(5)

Key Points

  • Modules cannot be instantiated like classes.
  • They are used with include, extend, or prepend.
  • Support code reuse without inheritance.
  • Provide namespaces to avoid conflicts.

Conclusion

Modules are a powerful feature in Ruby for organizing code, sharing functionality across classes, and maintaining clean namespaces. Mixins allow classes to use module methods without creating rigid inheritance hierarchies.
