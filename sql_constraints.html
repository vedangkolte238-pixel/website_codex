SQL Constraints

Rules to maintain data accuracy and integrity

What are SQL Constraints?

SQL constraints are rules applied to table columns to ensure valid and reliable data. They restrict the type of data that can be stored in a column.

1. NOT NULL

Ensures that a column cannot have NULL values.

CREATE TABLE students ( id INT NOT NULL, name VARCHAR(50) NOT NULL );

2. UNIQUE

Ensures all values in a column are unique.

CREATE TABLE users ( email VARCHAR(100) UNIQUE );

3. PRIMARY KEY

Uniquely identifies each record in a table.

CREATE TABLE employees ( emp_id INT PRIMARY KEY, emp_name VARCHAR(50) );

4. FOREIGN KEY

Creates a link between two tables.

CREATE TABLE orders ( order_id INT PRIMARY KEY, customer_id INT, FOREIGN KEY (customer_id) REFERENCES customers(customer_id) );

5. CHECK

Ensures that values meet a specific condition.

CREATE TABLE products ( price DECIMAL(10,2) CHECK (price > 0) );

6. DEFAULT

Assigns a default value if no value is provided.

CREATE TABLE accounts ( status VARCHAR(20) DEFAULT 'Active' );

Conclusion

SQL constraints help maintain data integrity and consistency. They are essential for designing reliable databases.
