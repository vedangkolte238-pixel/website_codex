Go Interfaces

Understanding Interfaces in Go

What is an Interface?

An interface in Go defines a set of method signatures. Any type that implements those methods automatically satisfies the interface.

Defining an Interface

type Shape interface { Area() float64 Perimeter() float64 }

Implementing an Interface

A struct implements an interface by defining all its methods.

type Rectangle struct { Width float64 Height float64 } func (r Rectangle) Area() float64 { return r.Width * r.Height } func (r Rectangle) Perimeter() float64 { return 2 * (r.Width + r.Height) }

Using an Interface

func printShape(s Shape) { fmt.Println("Area:", s.Area()) fmt.Println("Perimeter:", s.Perimeter()) }

Interface Example

func main() { rect := Rectangle{Width: 10, Height: 5} printShape(rect) }

Empty Interface

The empty interface can hold values of any type.

var data interface{} data = 10 data = "Hello"

Type Assertion

value, ok := data.(string) if ok { fmt.Println(value) }

Why Interfaces are Powerful

  • No explicit implementation required.
  • Promotes loose coupling.
  • Supports polymorphism.
  • Makes code flexible and testable.

Key Points

  • Interfaces define behavior, not data.
  • Implementation is implicit in Go.
  • Multiple types can satisfy the same interface.

Conclusion

Interfaces are a cornerstone of Go’s design philosophy. They allow you to write flexible, modular, and reusable code.
