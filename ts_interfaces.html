TypeScript Interfaces

Define the Structure of Objects

What is an Interface?

An interface in TypeScript is used to define the shape of an object. It describes what properties and methods an object should have, providing type safety.

Basic Interface Example

interface Person { name: string; age: number; } let user: Person = { name: "Alice", age: 25 };

Optional Properties

Use ? to make a property optional.

interface Person { name: string; age?: number; } let user1: Person = { name: "Bob" }; let user2: Person = { name: "Alice", age: 30 };

Readonly Properties

Use readonly to make a property immutable after initialization.

interface Person { readonly id: number; name: string; } let user: Person = { id: 101, name: "Alice" }; user.name = "Bob"; // allowed // user.id = 102; // Error: Cannot assign to 'id' because it is a read-only property

Methods in Interfaces

interface Person { name: string; greet(): string; } let user: Person = { name: "Alice", greet() { return `Hello, ${this.name}`; } }; console.log(user.greet());

Extending Interfaces

Interfaces can extend other interfaces to inherit properties.

interface Person { name: string; age: number; } interface Employee extends Person { employeeId: number; } let emp: Employee = { name: "Bob", age: 28, employeeId: 101 };

Conclusion

Interfaces help you enforce consistent object structures in TypeScript. They are especially useful in large applications for type safety and better code maintainability.
