HTML Semantic Tags

Meaningful structure for web pages

Introduction

Semantic tags clearly describe the meaning of content to both the browser and developers. They improve readability, accessibility, and SEO.

Why Use Semantic Tags

  • Better code readability
  • Improved accessibility
  • Search engine friendly

Common Semantic Tags

<header>

<header> <h1>Website Title</h1> </header>

<nav>

<nav> <ul> <li>Home</li> <li>About</li> </ul> </nav>

<main>

<main> <p>Main content goes here</p> </main>

<section>

<section> <h2>Section Title</h2> <p>Section content</p> </section>

<article>

<article> <h2>Article Title</h2> <p>Article content</p> </article>

<aside>

<aside> <p>Related information</p> </aside>

<footer>

<footer> <p>Copyright information</p> </footer>

Semantic vs Non-Semantic

Non-semantic tags like <div> and <span> do not describe their content, while semantic tags explain the purpose of the content.

Important Semantic Tags List

  • <header>
  • <nav>
  • <main>
  • <section>
  • <article>
  • <aside>
  • <footer>

Conclusion

Using semantic tags makes HTML code clean, meaningful, and accessible. They are a best practice in modern web development.
