Understanding Graph Data Structures in DSA

What Is a Graph?

A graph is a collection of nodes (called vertices) and edges connecting pairs of nodes. Graphs are used to represent relationships between objects.

Key Terminology

  • Vertex (Node): A fundamental unit of a graph
  • Edge: Connection between two vertices
  • Adjacent: Two vertices connected by an edge
  • Degree: Number of edges connected to a vertex

Types of Graphs

  • Directed Graph (Digraph): Edges have a direction
  • Undirected Graph: Edges have no direction
  • Weighted Graph: Edges carry a weight/value
  • Unweighted Graph: All edges are equal
  • Cyclic / Acyclic: Graph contains cycles or not

Graph Representation

  • Adjacency Matrix: 2D array representing connections between vertices
  • Adjacency List: List of lists to store connected vertices

Graph Traversal

  • Breadth-First Search (BFS): Visits nodes level by level
  • Depth-First Search (DFS): Visits nodes by exploring as far as possible along each branch

Example: Adjacency List in C++

#include <iostream> #include <vector> using namespace std; int main() { int vertices = 5; vector<int> adj[vertices]; // Adding edges adj[0].push_back(1); adj[0].push_back(2); adj[1].push_back(2); adj[2].push_back(0); adj[2].push_back(3); // Print adjacency list for(int i=0; i<vertices; i++) { cout << i << " : "; for(int v : adj[i]) cout << v << " "; cout << endl; } return 0; }

Applications of Graphs

  • Social networks (friends, followers)
  • Web page linking (PageRank)
  • Computer networks and routing
  • Maps and navigation systems
  • Dependency resolution in tasks and projects

Advantages of Graphs

  • Flexible representation of complex relationships
  • Supports many important algorithms (shortest path, MST)

Conclusion

Graphs are versatile data structures that can model complex relationships in real-world problems. Understanding their types, representations, and traversal algorithms is key for solving advanced DSA problems.
