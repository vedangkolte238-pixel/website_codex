JavaScript Async / Await

Write Asynchronous Code Like Synchronous

What is Async / Await?

Async and await are keywords used to work with Promises. They make asynchronous code easier to read and write.

Async Function

The async keyword is used before a function. An async function always returns a Promise.

async function greet() { return "Hello World"; } greet().then(function(message) { console.log(message); });

Await Keyword

The await keyword pauses execution until the Promise is resolved. It can only be used inside async functions.

function delay() { return new Promise(function(resolve) { setTimeout(function() { resolve("Done after 2 seconds"); }, 2000); }); } async function showResult() { let result = await delay(); console.log(result); } showResult();

Async / Await with Multiple Promises

function getNumber() { return new Promise(function(resolve) { resolve(5); }); } async function calculate() { let num = await getNumber(); let result = num * 2; console.log(result); } calculate();

Error Handling with try...catch

Errors in async functions are handled using try and catch.

function getData() { return new Promise(function(resolve, reject) { reject("Something went wrong"); }); } async function fetchData() { try { let data = await getData(); console.log(data); } catch (error) { console.log(error); } } fetchData();

Why Use Async / Await?

  • Cleaner and readable code
  • Easy error handling
  • No callback nesting
  • Used in modern JavaScript apps

Conclusion

Async and await simplify working with asynchronous code. They are built on Promises and widely used today.
