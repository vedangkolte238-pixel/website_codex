Java Polymorphism

One Method, Many Forms

What is Polymorphism?

Polymorphism means "many forms". In Java, it allows the same method name to perform different actions.

Why Use Polymorphism?

  • Improves code flexibility
  • Supports method overriding
  • Reduces code complexity
  • Makes programs scalable

Types of Polymorphism in Java

  • Compile-time Polymorphism (Method Overloading)
  • Runtime Polymorphism (Method Overriding)

Compile-time Polymorphism

Achieved using method overloading. Same method name with different parameters.

Method Overloading Example

class MathOperation { int add(int a, int b) { return a + b; } double add(double a, double b) { return a + b; } }

Runtime Polymorphism

Achieved using method overriding. Method call is decided at runtime.

Method Overriding Example

class Animal { void sound() { System.out.println("Animal makes a sound"); } } class Dog extends Animal { void sound() { System.out.println("Dog barks"); } }

Runtime Polymorphism with Object Reference

class Main { public static void main(String[] args) { Animal a = new Dog(); a.sound(); } }

Output

Dog barks

Key Points

  • Method overloading happens at compile time
  • Method overriding happens at runtime
  • Uses inheritance
  • Method overriding requires same method name and parameters

Rules for Method Overriding

  • Method name must be same
  • Parameter list must be same
  • Inheritance is required
  • Access level should not be reduced

Real-life Example

A person can be a student, employee, or customer. Same person behaves differently in different roles.

Conclusion

Polymorphism allows Java programs to behave differently using the same method. It improves readability, flexibility, and reusability of code.
