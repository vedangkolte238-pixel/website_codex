Understanding Object-Oriented Programming in C#

What is OOP?

OOP (Object-Oriented Programming) is a programming paradigm that uses objects and classes to design software. It helps in organizing code, reusing components, and modeling real-world problems.

Core OOP Concepts in C#

1. Classes and Objects

A class is a blueprint for creating objects. An object is an instance of a class.

class Car { public string brand; public void Start() { Console.WriteLine("Car started"); } } Car myCar = new Car(); myCar.brand = "Toyota"; myCar.Start();

2. Inheritance

Inheritance allows a class (child) to acquire properties and methods of another class (parent). It promotes code reusability.

class Vehicle { public void Start() { Console.WriteLine("Vehicle started"); } } class Car : Vehicle { public void Honk() { Console.WriteLine("Car honks"); } }

3. Polymorphism

Polymorphism allows methods to take many forms. It can be compile-time (method overloading) or run-time (method overriding).

// Method Overloading class Calculator { public int Add(int a, int b) { return a + b; } public double Add(double a, double b) { return a + b; } }

4. Encapsulation

Encapsulation is the practice of hiding data and providing access through methods (getters and setters).

class Person { private string name; public void SetName(string n) { name = n; } public string GetName() { return name; } }

5. Abstraction

Abstraction hides the complex implementation and shows only the necessary functionality to the user. It can be achieved using abstract classes or interfaces.

abstract class Shape { public abstract void Draw(); } class Circle : Shape { public override void Draw() { Console.WriteLine("Drawing Circle"); } }

Conclusion

OOP is the backbone of modern C# programming. Mastering these concepts helps you write clean, reusable, and maintainable code.
