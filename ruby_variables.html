Ruby Variables

Storing and Using Data in Ruby

What are Variables?

Variables in Ruby are used to store values which can be referenced and manipulated throughout the program. Ruby is dynamically typed, so variable types are inferred automatically.

Variable Naming Rules

  • Names must begin with a lowercase letter or underscore (_).
  • Names can contain letters, numbers, and underscores.
  • Case-sensitive: name and Name are different variables.
  • Cannot use reserved keywords (like def, class, etc.).

Variable Types

  • Local variables: start with a lowercase letter or underscore name = "Alice"
  • Instance variables: start with @ @age = 25
  • Class variables: start with @@ @@count = 0
  • Global variables: start with $ $app_name = "RubyApp"
  • Constants: start with an uppercase letter PI = 3.14

Examples

# Local variable name = "Alice" age = 30 # Instance variable @city = "New York" # Class variable @@users_count = 0 # Global variable $app_name = "MyRubyApp" # Constant PI = 3.14159 puts "Name: #{name}, Age: #{age}, City: #{@city}, App: #{$app_name}, PI: #{PI}"

Variable Scope

  • Local variables are accessible only within the block or method they are defined in.
  • Instance variables are accessible within the object they belong to.
  • Class variables are shared across all instances of a class.
  • Global variables are accessible from anywhere in the program.
  • Constants are accessible throughout the program but can be changed with a warning.

Conclusion

Variables are fundamental in Ruby for storing and manipulating data. Understanding variable types and scope helps in writing clean and maintainable code.
