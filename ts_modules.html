TypeScript Modules

Organize Your Code into Reusable Units

What are Modules?

Modules in TypeScript allow you to split code into separate files and import/export functionality between them. This helps in organizing code, improving maintainability, and avoiding naming conflicts.

Exporting from a Module

Use export to make variables, functions, or classes available outside the module.

// file: mathUtils.ts export function add(x: number, y: number): number { return x + y; } export const PI = 3.14; export class Calculator { multiply(a: number, b: number): number { return a * b; } }

Importing a Module

Use import to bring exported members into another file.

// file: main.ts import { add, PI, Calculator } from './mathUtils'; console.log(add(5, 3)); // 8 console.log(PI); // 3.14 let calc = new Calculator(); console.log(calc.multiply(4, 5)); // 20

Default Exports

You can export one default member from a module.

// file: message.ts export default function greet(name: string): string { return `Hello, ${name}`; } // file: main.ts import greet from './message'; console.log(greet("Alice")); // Hello, Alice

Re-exporting Modules

You can re-export members from other modules.

// file: utilities.ts export { add, Calculator } from './mathUtils';

Conclusion

Modules in TypeScript help you organize code into smaller, reusable, and maintainable units. Using import and export, you can share functionality across multiple files efficiently.
