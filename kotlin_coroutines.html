Kotlin Coroutines

Asynchronous Programming Made Simple

What are Coroutines?

Coroutines are a Kotlin feature used for writing asynchronous and non-blocking code in a simple and readable way. They allow long-running tasks to run without blocking the main thread.

Why Use Coroutines?

  • Simpler than threads
  • Efficient and lightweight
  • Easy to read and maintain
  • Perfect for network and background tasks

Coroutine Basics

Coroutines run inside a CoroutineScope and are launched using coroutine builders.

launch

import kotlinx.coroutines.* fun main() = runBlocking { launch { println("Coroutine running") } }

runBlocking

Blocks the current thread until coroutine execution is complete. Mostly used in testing and main functions.

runBlocking { delay(1000) println("Task completed") }

suspend Function

Suspend functions can pause execution without blocking the thread.

suspend fun fetchData() { delay(2000) println("Data fetched") }

delay()

Non-blocking alternative to Thread.sleep().

delay(1000) // waits for 1 second

async and await

Used when you want to return a result from a coroutine.

val result = async { delay(1000) "Hello Coroutines" } println(result.await())

Dispatchers

Dispatchers decide which thread the coroutine runs on.

  • Dispatchers.Main – UI thread
  • Dispatchers.IO – Network and disk operations
  • Dispatchers.Default – CPU intensive tasks
launch(Dispatchers.IO) { println("Running in IO thread") }
