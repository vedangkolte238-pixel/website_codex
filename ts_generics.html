TypeScript Generics

Write Reusable and Type-Safe Code

What are Generics?

Generics allow you to write functions, classes, or interfaces that work with multiple types while maintaining type safety.

Generic Function

function identity(value: T): T { return value; } let num = identity(10); let str = identity("Hello"); console.log(num); // 10 console.log(str); // Hello

Generic Array

You can define arrays with generics to enforce types.

function getArray(items: T[]): T[] { return new Array().concat(items); } let numberArray = getArray([1, 2, 3]); let stringArray = getArray(["A", "B", "C"]);

Generic Interface

interface KeyValuePair { key: K; value: V; } let kvp: KeyValuePair = { key: 1, value: "Apple" };

Generic Class

class GenericNumber { zeroValue: T; add: (x: T, y: T) => T; } let myNumber = new GenericNumber(); myNumber.zeroValue = 0; myNumber.add = (x, y) => x + y; console.log(myNumber.add(5, 10)); // 15

Constraints in Generics

Restrict the types allowed for a generic parameter.

interface Lengthwise { length: number; } function logLength(item: T): void { console.log(item.length); } logLength("Hello"); // 5 logLength([1,2,3]); // 3 // logLength(123); // Error: number has no 'length'

Conclusion

Generics make your TypeScript code more flexible and reusable while still providing type safety. They are especially useful when working with collections or utility functions.
