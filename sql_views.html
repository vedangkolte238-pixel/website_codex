SQL Views

Virtual tables to simplify complex queries

What is a View?

A view is a virtual table based on the result of a SQL query. It does not store data physically, but displays data from one or more tables.

1. Creating a View

Use the CREATE VIEW statement to define a view.

CREATE VIEW employee_view AS SELECT emp_id, emp_name, department FROM employees WHERE status = 'Active';

2. Using a View

You can query a view like a normal table.

SELECT * FROM employee_view;

3. Updating a View

Use the CREATE OR REPLACE VIEW statement to update a view.

CREATE OR REPLACE VIEW employee_view AS SELECT emp_id, emp_name, department, salary FROM employees WHERE status = 'Active';

4. Dropping a View

Use the DROP VIEW statement to delete a view.

DROP VIEW employee_view;

Benefits of Views

  • Simplify complex queries
  • Provide a level of security by restricting columns
  • Reuse queries easily
  • Can combine data from multiple tables

Conclusion

SQL views are useful for abstraction and security. They make querying easier without modifying the underlying tables.
