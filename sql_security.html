SQL Security

Protecting data and controlling access in databases

What is SQL Security?

SQL security involves measures to protect databases from unauthorized access, data breaches, and misuse. It ensures that only authorized users can perform allowed operations.

1. User Authentication

Every user must have a unique login and password to access the database.

CREATE USER 'john'@'localhost' IDENTIFIED BY 'StrongPassword123';

2. Privileges and Roles

Grant specific privileges to users or roles to control their actions on the database.

GRANT SELECT, INSERT ON employees TO 'john'@'localhost'; REVOKE DELETE ON employees FROM 'john'@'localhost';

3. Views for Security

Use views to restrict access to sensitive columns or rows.

CREATE VIEW public_employees AS SELECT emp_id, name, department FROM employees;

4. Encryption

Encrypt sensitive data such as passwords or personal information.

-- Example: Store password as a hash INSERT INTO users(username, password) VALUES ('john', SHA2('MyPassword', 256));

5. SQL Injection Prevention

Avoid dynamic queries with user input. Use prepared statements and parameterized queries.

-- Example in MySQL PREPARE stmt FROM 'SELECT * FROM users WHERE username=?'; SET @user='john'; EXECUTE stmt USING @user;

6. Auditing and Logging

Track database operations to detect unauthorized access or suspicious activities.

CREATE TRIGGER log_insert AFTER INSERT ON employees FOR EACH ROW BEGIN INSERT INTO audit_log(user_id, action, timestamp) VALUES (CURRENT_USER(), 'INSERT', NOW()); END;

Conclusion

SQL security is critical for protecting sensitive data and maintaining trust. Implement user access controls, encryption, and monitoring to safeguard your database.
