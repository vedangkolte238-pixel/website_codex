Swift Protocols

Defining Blueprints for Methods and Properties

What Is a Protocol?

A protocol defines a blueprint of methods, properties, or other requirements that a class, struct, or enum can adopt. Protocols help enforce consistency and enable polymorphism.

Defining a Protocol

protocol Vehicle { var name: String { get } func start() }

Adopting a Protocol

A class or struct adopts a protocol and implements its requirements.

struct Car: Vehicle { var name: String func start() { print("\(name) is starting") } } let myCar = Car(name: "Tesla") myCar.start()

Protocols with Properties

Protocols can require properties to be gettable or settable.

protocol PersonProtocol { var fullName: String { get set } } struct Student: PersonProtocol { var fullName: String }

Protocol Inheritance

A protocol can inherit from one or more other protocols.

protocol Flying { func fly() } protocol Bird: Flying { func chirp() } struct Sparrow: Bird { func fly() { print("Sparrow is flying") } func chirp() { print("Chirp chirp") } }

Protocol as a Type

Protocols can be used as types for variables, arrays, or function parameters.

let vehicle: Vehicle = Car(name: "BMW") vehicle.start()

Protocol Extensions

You can provide default implementations of methods using protocol extensions.

extension Vehicle { func start() { print("\(name) is ready to go") } }

Conclusion

Protocols in Swift provide a flexible way to define interfaces and enforce requirements across classes, structs, and enums. They are key to writing clean, modular, and reusable code.
