Go Concurrency

Writing Concurrent Programs in Go

What is Concurrency?

Concurrency means executing multiple tasks at the same time. Go provides built-in support for concurrency, making it easy to write fast and efficient programs.

Goroutines

A goroutine is a lightweight thread managed by the Go runtime. You create a goroutine using the go keyword.

go functionName()

Example

func sayHello() { fmt.Println("Hello from goroutine") } func main() { go sayHello() fmt.Println("Hello from main") }

Channels

Channels are used to communicate between goroutines. They help synchronize concurrent tasks.

ch := make(chan int)

Sending and Receiving Data

ch <- 10 // send value := <-ch // receive

Channel Example

func sendData(ch chan int) { ch <- 42 } func main() { ch := make(chan int) go sendData(ch) fmt.Println(<-ch) }

Buffered Channels

Buffered channels allow sending data without immediate receiving.

ch := make(chan int, 3) ch <- 1 ch <- 2 ch <- 3

Select Statement

The select statement lets a goroutine wait on multiple channels.

select { case msg := <-ch1: fmt.Println(msg) case msg := <-ch2: fmt.Println(msg) }

WaitGroup

WaitGroup is used to wait for multiple goroutines to finish.

var wg sync.WaitGroup wg.Add(1) go func() { defer wg.Done() fmt.Println("Task done") }() wg.Wait()

Why Use Go Concurrency?

  • Simple and readable syntax
  • Efficient use of CPU cores
  • Built-in concurrency support
  • Safe communication using channels

Key Points

  • Goroutines are lightweight and fast.
  • Channels enable safe communication.
  • Select helps manage multiple channels.
  • Concurrency is not the same as parallelism.

Conclusion

Go’s concurrency model is one of its strongest features. With goroutines and channels, you can write powerful concurrent programs with minimal complexity.
