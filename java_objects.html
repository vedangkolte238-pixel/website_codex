Java Objects

Instances of Java Classes

What is an Object?

An object is a real-world entity created from a class. It represents data and behavior together.

Why Objects are Important

  • Objects represent real-life entities
  • They help structure programs
  • They allow reuse of class code
  • They support object-oriented programming

Creating an Object

Objects are created using the new keyword.

ClassName objectName = new ClassName();

Example Class

class Car { String brand; int speed; }

Creating an Object of Car

Car c1 = new Car();

Assigning Values to Object

c1.brand = "BMW"; c1.speed = 120;

Accessing Object Data

System.out.println(c1.brand); System.out.println(c1.speed);

Object with Method

class Car { String brand; int speed; void display() { System.out.println(brand); System.out.println(speed); } }

Calling Method Using Object

Car c1 = new Car(); c1.brand = "Audi"; c1.speed = 150; c1.display();

Multiple Objects

Multiple objects can be created from the same class.

Car c1 = new Car(); Car c2 = new Car(); c1.brand = "BMW"; c2.brand = "Tesla";

Objects with Constructor

class Car { String brand; int speed; Car(String b, int s) { brand = b; speed = s; } }

Creating Object Using Constructor

Car c1 = new Car("BMW", 120); Car c2 = new Car("Tesla", 140);

Object Reference

The variable stores the reference of the object, not the object itself.

Car c1 = new Car(); Car c2 = c1;

Object and Memory

  • Objects are stored in heap memory
  • Reference variables are stored in stack
  • Garbage collector removes unused objects

Key Points

  • Object is an instance of a class
  • Created using new keyword
  • Accessed using dot operator
  • Multiple objects can exist for one class

Conclusion

Objects make Java powerful by combining data and behavior. Understanding objects is essential for mastering Java OOP concepts.
