Rust Ownership

Memory Safety and the Ownership System

What is Ownership?

Ownership is a key feature in Rust that ensures memory safety without a garbage collector. It follows three rules:

  • Each value has a variable called its owner.
  • There can only be one owner at a time.
  • When the owner goes out of scope, the value is automatically dropped.

Basic Ownership Example

fn main() { let s1 = String::from("Hello"); let s2 = s1; // Ownership moves to s2 // println!("{}", s1); // ERROR: s1 is no longer valid println!("{}", s2); // Works }

Borrowing

Rust allows references to a value without taking ownership. This is called borrowing.

fn main() { let s = String::from("Hello"); // Borrow s as immutable reference let len = calculate_length(&s); println!("Length of '{}' is {}", s, len); } fn calculate_length(s: &String) -> usize { s.len() }

Mutable Borrowing

fn main() { let mut s = String::from("Hello"); // Mutable borrow to modify value change(&mut s); println!("{}", s); } fn change(some_string: &mut String) { some_string.push_str(", World!"); }

Rules of References

  • You can have any number of immutable references (&) to a value at a time.
  • You can have only one mutable reference (&mut) at a time.
  • Mutable and immutable references cannot coexist.

Applications

  • Memory-safe programming without garbage collection
  • Preventing data races at compile time
  • Efficient and predictable resource management

Conclusion

Ownership is the cornerstone of Rust’s memory safety model. Understanding ownership, borrowing, and references is essential to write safe and efficient Rust programs.
