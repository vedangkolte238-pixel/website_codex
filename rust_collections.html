Rust Collections

Working with Groups of Data

Introduction

Rust provides several standard collection types for storing and managing groups of values. Common collections include Vector, String, HashMap, and HashSet.

Vectors (Vec)

Vectors are resizable arrays.

fn main() { let mut numbers: Vec = Vec::new(); numbers.push(10); numbers.push(20); numbers.push(30); for num in &numbers { println!("{}", num); } println!("Second element: {}", numbers[1]); }

Strings

Rust has two main string types: String (growable, heap-allocated) and &str (string slice).

fn main() { let mut s = String::from("Hello"); s.push_str(", Rust!"); println!("{}", s); let slice: &str = &s[0..5]; println!("Slice: {}", slice); }

HashMap

HashMap stores key-value pairs.

use std::collections::HashMap; fn main() { let mut scores = HashMap::new(); scores.insert("Alice", 50); scores.insert("Bob", 60); for (key, value) in &scores { println!("{}: {}", key, value); } println!("Alice's score: {}", scores.get("Alice").unwrap()); }

HashSet

HashSet stores unique values without duplicates.

use std::collections::HashSet; fn main() { let mut fruits = HashSet::new(); fruits.insert("Apple"); fruits.insert("Banana"); fruits.insert("Apple"); // duplicate ignored for fruit in &fruits { println!("{}", fruit); } }

Applications

  • Storing ordered or unordered collections of data
  • Counting frequencies or removing duplicates
  • Associating keys with values (like a dictionary)
  • Efficiently iterating and searching datasets

Conclusion

Rust’s standard collections provide powerful and efficient ways to manage groups of values while adhering to ownership and borrowing rules. Choosing the right collection type is important for performance and correctness.
