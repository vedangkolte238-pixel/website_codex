Ruby Exception Handling

Handling Errors Safely in Ruby

Introduction

Exception handling allows a program to deal with runtime errors gracefully instead of crashing. Ruby provides a clean and readable way to handle errors using begin and rescue.

Basic Exception Handling

begin result = 10 / 0 rescue puts "An error occurred" end

Rescuing Specific Exceptions

begin num = Integer("abc") rescue ArgumentError puts "Invalid number format" end

Using Else

The else block runs only if no exception occurs.

begin result = 10 / 2 rescue ZeroDivisionError puts "Cannot divide by zero" else puts "Result is #{result}" end

Ensure Block

The ensure block always runs, whether an exception occurs or not.

begin file = File.open("test.txt") puts file.read rescue puts "File error" ensure file.close if file end

Raising Exceptions

You can manually raise an exception using raise.

def check_age(age) raise "Age must be positive" if age < 0 puts "Age is valid" end check_age(-5)

Retry Keyword

The retry keyword retries the code inside the begin block.

attempts = 0 begin attempts += 1 puts "Attempt #{attempts}" raise "Error" rescue retry if attempts < 3 end

Common Ruby Exceptions

  • StandardError
  • ArgumentError
  • NoMethodError
  • TypeError
  • ZeroDivisionError
  • IOError

Conclusion

Proper exception handling makes Ruby applications more reliable and user-friendly. Using specific exceptions and cleanup logic ensures clean and predictable behavior.
