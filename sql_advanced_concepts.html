SQL Advanced Concepts

Advanced techniques for efficient and complex database operations

1. Subqueries and Nested Queries

Use queries inside other queries to filter, compute, or join data in complex ways.

SELECT name FROM employees WHERE salary > (SELECT AVG(salary) FROM employees);

2. Indexing

Indexes improve query performance by allowing fast lookups on large tables.

CREATE INDEX idx_emp_name ON employees(name);

3. Views

Virtual tables based on queries. Simplify complex joins and hide sensitive columns.

CREATE VIEW active_employees AS SELECT emp_id, name, department FROM employees WHERE status='Active';

4. Transactions

Group multiple queries into a single unit. Ensures ACID properties.

BEGIN TRANSACTION; UPDATE accounts SET balance = balance - 500 WHERE account_id = 1; UPDATE accounts SET balance = balance + 500 WHERE account_id = 2; COMMIT;

5. Stored Procedures

Predefined SQL scripts that perform specific tasks. Helps reuse logic.

CREATE PROCEDURE IncreaseSalary(emp INT, amount DECIMAL) BEGIN UPDATE employees SET salary = salary + amount WHERE emp_id = emp; END;

6. Triggers

Automatic actions that fire on INSERT, UPDATE, or DELETE events.

CREATE TRIGGER trg_after_insert AFTER INSERT ON employees FOR EACH ROW BEGIN INSERT INTO logs(emp_id, action) VALUES (NEW.emp_id, 'Inserted'); END;

7. CTE (Common Table Expressions)

Temporary result sets that can be referenced within a SELECT, INSERT, UPDATE, or DELETE statement.

WITH DeptCount AS ( SELECT department, COUNT(*) AS num_emp FROM employees GROUP BY department ) SELECT * FROM DeptCount WHERE num_emp > 5;

8. Window Functions

Perform calculations across a set of rows related to the current row without collapsing the result.

SELECT name, salary, RANK() OVER (PARTITION BY department ORDER BY salary DESC) AS dept_rank FROM employees;

Conclusion

SQL advanced concepts allow for optimized queries, better performance, and complex operations. Mastering these concepts helps build efficient and scalable databases.
