Java Exception Handling

Handling Runtime Errors Safely

What is an Exception?

An exception is an unwanted or unexpected event that occurs during program execution. It disrupts the normal flow of the program.

Why Exception Handling?

  • Prevents program termination
  • Improves program reliability
  • Handles runtime errors gracefully
  • Makes debugging easier

Common Exceptions

  • ArithmeticException
  • NullPointerException
  • ArrayIndexOutOfBoundsException
  • NumberFormatException

Exception Handling Keywords

  • try
  • catch
  • finally
  • throw
  • throws

try and catch

The try block contains code that may cause an exception. The catch block handles the exception.

try { int a = 10 / 0; } catch (ArithmeticException e) { System.out.println("Cannot divide by zero"); }

Output

Cannot divide by zero

Multiple catch Blocks

try { int arr[] = new int[5]; arr[10] = 50; } catch (ArithmeticException e) { System.out.println("Arithmetic Error"); } catch (ArrayIndexOutOfBoundsException e) { System.out.println("Array index error"); }

finally Block

The finally block always executes whether an exception occurs or not.

try { int x = 20 / 2; } catch (Exception e) { System.out.println("Error occurred"); } finally { System.out.println("Program finished"); }

throw Keyword

The throw keyword is used to explicitly throw an exception.

if (age < 18) { throw new ArithmeticException("Not eligible to vote"); }

throws Keyword

The throws keyword is used in method declaration to pass exception responsibility.

void readFile() throws IOException { // file handling code }

Checked vs Unchecked Exceptions

  • Checked: Checked at compile time (IOException)
  • Unchecked: Checked at runtime (ArithmeticException)

Custom Exception

User-defined exceptions can be created by extending the Exception class.

class MyException extends Exception { MyException(String msg) { super(msg); } }

Key Points

  • Exceptions occur at runtime
  • Use try-catch to handle errors
  • finally always executes
  • Custom exceptions improve clarity

Conclusion

Exception handling makes Java programs more robust and user-friendly. It is an essential concept for real-world applications.
