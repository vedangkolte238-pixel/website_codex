Go Packages

Organizing Code Using Packages in Go

What is a Package?

A package in Go is a collection of related Go source files. Packages help organize code, improve readability, and promote reuse.

Main Package

Every Go program starts execution from the main package.

package main func main() { fmt.Println("Hello Go") }

Importing Packages

Packages are imported using the import keyword.

import "fmt"

Multiple Imports

import ( "fmt" "math" )

Standard Library Packages

  • fmt – Input and output
  • math – Mathematical operations
  • time – Date and time
  • os – Operating system functions

Creating Your Own Package

package calculator func Add(a int, b int) int { return a + b }

Using a Custom Package

import "calculator" func main() { result := calculator.Add(5, 3) fmt.Println(result) }

Exported vs Unexported Names

Identifiers starting with a capital letter are exported and accessible outside the package.

func PublicFunc() {} func privateFunc() {}

Package Initialization

The init() function runs automatically before main().

func init() { fmt.Println("Package initialized") }

Go Modules

Go modules manage dependencies in Go projects.

go mod init myproject

Key Points

  • Packages organize and structure Go code.
  • The main package is the entry point.
  • Capital letters control visibility.
  • Modules manage dependencies.

Conclusion

Packages are essential for building scalable Go applications. They help keep code clean, reusable, and easy to maintain.
