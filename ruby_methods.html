Ruby Data Types

Understanding Ruby's Core Data Types

Introduction

Ruby is dynamically typed, meaning variables don’t need an explicit type declaration. Data types are inferred from the value assigned. Here are the main Ruby data types.

Numeric Types

Used for integer and floating-point numbers.

# Integer a = 10 b = -5 # Float c = 3.14 d = -0.99

String

Represents sequences of characters.

name = "Alice" greeting = 'Hello, world!' puts "#{greeting}, #{name}"

Symbol

Immutable, reusable identifier used as keys or constants.

status = :active puts status

Boolean

Represents truth values: true or false.

is_active = true is_admin = false

Nil

Represents the absence of a value (nil).

data = nil puts data.nil? # true

Array

Ordered collection of objects.

numbers = [1, 2, 3, 4] fruits = ["Apple", "Banana", "Cherry"]

Hash

Key-value pairs, similar to dictionaries.

person = { "name" => "Alice", "age" => 25 } puts person["name"] # Alice

Conclusion

Understanding Ruby’s data types is essential for writing effective programs. Everything in Ruby is an object, so methods and properties can be used on all data types.
