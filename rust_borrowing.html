Rust Borrowing

Accessing Data Without Taking Ownership

What is Borrowing?

Borrowing in Rust allows a function or variable to access a value without taking ownership of it. This prevents unnecessary copying and ensures memory safety.

Immutable Borrowing

fn main() { let s = String::from("Hello"); // Borrow s immutably let len = calculate_length(&s); println!("The length of '{}' is {}", s, len); } fn calculate_length(s: &String) -> usize { s.len() }

Here, &s creates an immutable reference, allowing read-only access to s without transferring ownership.

Mutable Borrowing

fn main() { let mut s = String::from("Hello"); // Borrow s mutably change(&mut s); println!("{}", s); } fn change(some_string: &mut String) { some_string.push_str(", World!"); }

Using &mut s, we can modify the original value through the reference. Only one mutable reference is allowed at a time.

Rules of Borrowing

  • You can have multiple immutable references to a value simultaneously.
  • Only one mutable reference is allowed at a time.
  • Mutable and immutable references cannot coexist in the same scope.
  • References must always be valid for the duration of use.

Applications

  • Safe access to values without transferring ownership
  • Efficient memory usage
  • Preventing dangling references and data races

Conclusion

Borrowing is a core concept in Rust that allows safe and efficient access to data. Understanding immutable and mutable borrowing is essential for writing safe Rust programs that adhere to the ownership model.
