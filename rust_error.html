Rust Error Handling

Handling Errors Safely in Rust

Introduction

Rust distinguishes between recoverable and unrecoverable errors. Recoverable errors use Result, while unrecoverable errors cause a panic!.

Option Type

The Option enum is used for values that may or may not exist.

fn main() { let some_number: Option = Some(10); let no_number: Option = None; match some_number { Some(x) => println!("Value: {}", x), None => println!("No value"), } }

Result Type

Result is used for recoverable errors with Ok and Err variants.

use std::fs::File; use std::io::ErrorKind; fn main() { let file = File::open("hello.txt"); let file = match file { Ok(f) => f, Err(ref e) if e.kind() == ErrorKind::NotFound => { println!("File not found, creating a new one..."); File::create("hello.txt").unwrap() } Err(e) => panic!("Problem opening the file: {:?}", e), }; }

Panic!

Unrecoverable errors cause the program to stop immediately using panic!.

fn main() { let v = vec![1, 2, 3]; v[10]; // This will panic because index is out of bounds }

Using unwrap and expect

let f = File::open("hello.txt").unwrap(); // panics if error occurs let f = File::open("hello.txt").expect("File not found!"); // custom panic message

Applications

  • Safe handling of file operations and I/O
  • Graceful handling of optional values
  • Preventing program crashes by using Result instead of panics
  • Debugging with informative panic messages when necessary

Conclusion

Rust’s error handling ensures safe and predictable behavior. Understanding Option, Result, and panic! is essential for writing robust Rust programs.
